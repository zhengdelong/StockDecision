# 2万元本金盈利导向交易策略

## 策略定位

本策略以盈利为目标，风险控制是约束，不是最终目的。策略核心是用 2 万元本金集中参与少数高质量机会。

目标函数：

```text
最大化扣除交易成本后的净利润
约束最大回撤、单笔亏损、交易频率和执行纪律
```

策略不追求每天都有交易。没有高质量机会时，空仓是正常状态。

## 市场环境判断

先判断市场是否有赚钱效应。只有市场环境允许时，才生成买入计划。

观察指数：

- 沪深300。
- 中证500。
- 创业板指。

每个指数满足以下条件记为 1 个有效市场确认：

```text
指数收盘价 > 指数 MA20
且 指数 MA20 最近 5 个交易日向上
```

交易权限：

| 有效市场确认数 | 市场状态 | 交易动作 |
|---:|---|---|
| 3 | 强市场 | 可持有 2 只股票，总仓位可到 100% |
| 2 | 可交易市场 | 可持有 1 到 2 只股票，总仓位不超过 80% |
| 1 | 弱机会市场 | 只允许 1 只股票，总仓位不超过 50% |
| 0 | 禁止开仓 | 不开新仓，只处理持仓 |

## 股票池硬过滤

每天先做硬过滤，不满足则不进入评分。

| 过滤项 | 默认规则 | 原因 |
|---|---|---|
| 市场范围 | 沪深主板 + 创业板 | 避免过早覆盖复杂品种 |
| 风险标签 | 排除 ST、*ST、退市整理 | 避免高退市风险 |
| 上市时间 | 上市满 250 个交易日 | 避免新股数据不足和波动异常 |
| 股价 | 5 元到 80 元 | 适配 2 万本金和 100 股交易单位 |
| 流动性 | 20 日平均成交额 >= 2 亿元 | 避免买卖困难 |
| 趋势底线 | 收盘价 > MA60 | 不买中期趋势偏弱股票 |
| 波动 | ATR14 / 收盘价 <= 7% | 避免波动过大 |
| 短期涨幅 | 最近 10 日涨幅 <= 30% | 避免追极端过热 |
| 估值 | PE > 0，PB <= 8 | 排除亏损和估值极端 |
| 财务 | 净利润同比 >= -30% | 排除明显恶化 |

## 评分模型

满分 100 分。低于 85 分不生成买入计划。

### 相对强度：30 分

| 指标 | 条件 | 分数 |
|---|---|---:|
| 20 日涨幅排名 | 位于全市场前 20% | 10 |
| 60 日涨幅排名 | 位于全市场前 20% | 8 |
| 跑赢沪深300 | 20 日涨幅高于沪深300 | 4 |
| 跑赢所属行业 | 20 日涨幅高于行业平均 | 5 |
| 未严重过热 | 10 日涨幅 <= 25% | 3 |

### 趋势结构：25 分

| 指标 | 条件 | 分数 |
|---|---|---:|
| 中期趋势 | 收盘价 > MA60 | 6 |
| 多头排列 | MA20 > MA60 > MA120 | 8 |
| 短期趋势 | 收盘价 > MA20 | 4 |
| MA20 斜率 | MA20 最近 10 日向上 | 4 |
| 不追高 | 收盘价距离 MA20 <= 10% | 3 |

### 量价确认：25 分

| 指标 | 条件 | 分数 |
|---|---|---:|
| 放量上涨 | 今日涨幅 > 2%，成交额 > 20 日均额 1.5 倍 | 8 |
| 突破平台 | 收盘价创 20 日新高 | 7 |
| 换手活跃 | 换手率 2% 到 8% | 4 |
| 资金持续 | 最近 5 日成交额均值 > 20 日成交额均值 | 4 |
| 无巨量滞涨 | 放量但不收长上影 | 2 |

### 基本面兜底：20 分

| 指标 | 条件 | 分数 |
|---|---|---:|
| 净利润增长 | 最近一期净利润同比 > 0 | 5 |
| 营收增长 | 最近一期营收同比 > 0 | 4 |
| ROE | ROE >= 8% | 5 |
| 估值不过分 | PE 0 到 80，PB 0 到 8 | 4 |
| 市值 | 流通市值 >= 50 亿 | 2 |

### 分级

| 总分 | 等级 | 动作 |
|---:|---|---|
| 90 以上 | A | 可买候选 |
| 85 到 89 | B | 强观察，等买点 |
| 80 到 84 | C | 学习观察，不交易 |
| 80 以下 | D | 淘汰 |

## 买入策略 A：强势突破

强势突破是主盈利策略，用于市场强、板块强、个股放量突破时。

### 触发条件

必须同时满足：

- 市场状态不是禁止开仓。
- 总分 >= 90。
- 所属行业 20 日涨幅排名全行业前 30%。
- 个股收盘价创 20 日新高。
- 今日成交额 >= 20 日平均成交额 * 1.5。
- 今日涨幅在 2% 到 7% 之间。
- 收盘价距离 MA20 不超过 10%。
- 当日不是放量长上影。

### 次日买入规则

```text
如果次日高开 <= 3%，且价格没有跌回突破价以下，允许买入。
如果次日高开 > 3%，放弃。
如果盘中跌回突破价以下，放弃。
```

### 仓位

```text
强市场：买入 10000 元。
可交易市场：买入 8000 元。
弱机会市场：不做突破，只观察。
```

## 买入策略 B：强势股回踩 MA20

回踩 MA20 是提高胜率的策略，用于强势股短期调整后重新转强。

### 触发条件

必须同时满足：

- 市场状态不是禁止开仓。
- 总分 >= 85。
- 过去 60 日涨幅排名全市场前 20%。
- 收盘价 > MA60。
- MA20 > MA60。
- 最近 5 日最低价接近 MA20，偏离不超过 3%。
- 今日收盘价重新站上 MA20。
- 今日成交额 >= 20 日平均成交额。
- 今日涨幅 <= 5%。

### 次日买入规则

```text
如果次日高开 <= 2%，且价格仍在 MA20 上方，允许买入。
如果次日高开 > 2%，放弃。
如果跌破 MA20，放弃。
```

### 仓位

```text
强市场：买入 10000 元。
可交易市场：买入 8000 元。
弱机会市场：买入 8000 元以内，且只能持有 1 只。
```

## 不买规则

满足任意一条则不生成买入计划：

- 市场状态为禁止开仓。
- 股票已涨停或预计无法正常买入。
- 次日高开超过策略阈值。
- 收盘价距离 MA20 超过 10%。
- 最近 10 日涨幅超过 30%。
- 当日放量长上影。
- 预期盈亏比低于 2。
- 已持仓数量达到上限。
- 当前总仓位达到上限。
- 财务数据严重缺失且无法完成基本排雷。

## 仓位规则

本金默认 20000 元。

| 项目 | 规则 |
|---|---|
| 最大持股数 | 2 只 |
| 单票金额 | 8000 到 10000 元 |
| 单笔最大亏损 | 400 元 |
| 强市场总仓位 | 最高 100% |
| 可交易市场总仓位 | 最高 80% |
| 弱机会市场总仓位 | 最高 50% |
| 禁止开仓市场 | 0 新仓 |

买入股数按 100 股整数倍向下取整：

```text
预估股数 = floor(买入金额 / 买入价 / 100) * 100
```

如果向下取整后买入金额低于 6000 元，放弃该股票，因为仓位太小会被交易成本放大。

## 止损规则

单笔最大亏损：

```text
最大亏损金额 = 400 元
```

资金止损价：

```text
资金止损价 = 买入价 - 最大亏损金额 / 买入股数
```

技术止损价：

```text
强势突破：突破价下方 3%。
MA20 回踩：MA20 下方 2%。
```

最终止损价：

```text
最终止损价 = max(资金止损价, 技术止损价)
```

如果收盘价跌破最终止损价，次日卖出。若盘中发生极端跳空，按次日可成交价格执行，并在复盘中记录滑点。

## 止盈规则

| 盈利状态 | 动作 |
|---|---|
| 浮盈 >= 8% | 止损上移到成本价 |
| 浮盈 >= 15% | 卖出 30% |
| 浮盈 >= 25% | 卖出 50% |
| 剩余仓位跌破 MA20 | 清仓 |
| 从最高收盘价回撤 10% | 清仓 |
| 放量长上影且次日不能修复 | 卖出 50% |

止盈原则：

```text
小亏及时认，大赚让趋势继续。
```

## 盈亏比规则

每条买入计划必须计算预期收益和最大亏损。

最大亏损：

```text
最大亏损 = (买入价 - 止损价) * 买入股数 + 预估交易成本
```

预期收益：

```text
强势突破目标价 = 买入价 * 1.12
MA20 回踩目标价 = 买入价 * 1.10
预期收益 = (目标价 - 买入价) * 买入股数 - 预估交易成本
```

盈亏比：

```text
盈亏比 = 预期收益 / 最大亏损
```

如果盈亏比 < 2，不生成买入计划，只展示为学习观察。

## 每日输出

系统每天输出三类股票：

- 可买：明天满足价格条件即可执行。
- 等回踩：趋势好，但距离 MA20 太远。
- 等突破：平台整理中，突破后再看。

每只股票必须输出：

```text
代码、名称、行业、总分、等级、策略类型、触发价、放弃条件、
建议买入金额、预估股数、止损价、最大亏损、目标价、预期收益、盈亏比、
为什么强、为什么现在可以买或不能买、主要风险、学习知识点
```
